# Constitution
# Mantis — Autonomous Vulnerability Discovery Harness

## What this is

A defensive security research tool modeled on the Anthropic Glasswing/Mythos scaffold. It autonomously finds, validates, and reports vulnerabilities in open-source software of all types — including C/C++, and extensible to other languages — using a provider-agnostic LLM agent loop (powered by litellm) as the agentic runtime inside isolated Docker containers. It is built for enterprise use in regulated industries including financial services and federal government, and must satisfy all relevant NIST frameworks and applicable regulatory requirements.

The system replicates the five-stage Anthropic methodology:
1. File-ranking pre-pass (LLM scores files by vulnerability likelihood)
2. Parallel isolated containers (one agent per file)
3. Agentic hypothesis-test loop with AddressSanitizer as task verifier
4. PoC generation and optional exploit attempt
5. Validation agent pass (separate LLM call filters noise)

## Non-negotiable principles

### P1 — Isolation is absolute
Every worker container has egress only to the configured model provider API endpoint(s). Default-deny, allowlist-based. No other outbound connections. No inter-container communication. No access to internal infrastructure. No package manager calls at runtime. The target binary may attempt network calls; the container network policy silently drops them. This is enforced at the Docker/Kubernetes network layer, not by trusting the agent.

### P2 — Human review before any external action
No finding, disclosure, patch, or report reaches any external party without explicit human sign-off recorded in the audit log with timestamp and reviewer identity. The harness produces a findings package for human consumption. It does not email, post, push, or transmit anything autonomously. This gate is not configurable, not bypassable, not optional.

### P3 — Every action is logged before it executes
All tool calls, all shell commands executed inside containers, all ASAN output, all LLM responses, all agent reasoning steps are written to the immutable audit log before any subsequent action. Log writes are synchronous. If the log write fails, the run fails. This produces the evidence trail required for regulatory compliance and governance platforms.

### P4 — No credential persistence in images
`ANTHROPIC_API_KEY` and any other secrets are injected at container runtime via environment variables. They are never baked into Docker images, never written to disk inside containers, never logged. Images are built credential-free and are safe to push to any registry.

### P5 — Containers are ephemeral and disposable
Worker containers are created fresh for each job and destroyed after result collection. No state persists inside containers between runs. Results are written to the external results store before container teardown. A container that crashes or times out produces a failed-job record, not data loss.

### P6 — Cost is tracked in real time
Every API call records input tokens, output tokens, model, and cost estimate. Running totals are visible in the dashboard and written to the audit log. Operators can set per-run and per-day spend limits that hard-stop new job dispatch when exceeded.

### P7 — The tool never decides severity alone
CVSS scoring is produced by the system as a starting estimate. A human security researcher must confirm or override severity before a finding enters the output package. The system may rank and prioritize but never makes final severity determinations.

### P8 — Exploit code is contained
Any exploit code generated by the agent exists only inside the container during its run and in the encrypted findings store afterward. Exploit code is never printed to stdout in plaintext, never written to shared volumes, never transmitted unencrypted. Access to the findings store requires authenticated, role-based access.

## Technology constraints

- **Language**: Python 3.12+ for orchestrator
- **Agent runtime**: litellm + custom ReAct agent loop (Python, provider-agnostic). Supports any litellm-compatible model provider (Anthropic, OpenAI, Google, Ollama, etc.)
- **Containers**: Docker (local dev), Kubernetes (production)
- **Queue**: Redis (local dev), can swap to Postgres for production
- **Target binaries**: C/C++ projects compiled with Clang + AddressSanitizer (`-fsanitize=address -g -O1`)
- **Model**: Configurable via litellm model strings (e.g. `anthropic/claude-opus-4-6`, `openai/gpt-4o`). Default: `anthropic/claude-opus-4-6` for all stages; overridable per stage
- **Audit store**: Append-only structured JSONL log, SHA-3 hash chained per entry. Designed for ingestion by governance platforms — the JSONL format and hash chain are the evidence artifact; no gateway service required in v1.
- **Results store**: Postgres with encrypted `findings` table; no plaintext exploit code columns

## Technology non-constraints (deliberately left open)
- Worker agent model is configurable — any litellm-supported model string. Default `anthropic/claude-opus-4-6`, overridable per run to any provider/model combination (e.g. `openai/gpt-4o`, `ollama/llama3`)
- Target language is C/C++ for v1; architecture supports adding Rust, Go, Java in future iterations
- Dashboard is read-only observability; implementation choice (simple Flask, or CLI-only) is left to implementer

## Success criteria for v1

A successful v1 harness:
1. Finds at least one confirmed true-positive vulnerability in a known-vulnerable OSS-Fuzz project (e.g. libpng, libtiff, sqlite) during local testing
2. Produces a structured bug report with file, line, crash type, CVSS estimate, minimal reproduction case, and candidate patch
3. Runs 4 parallel workers locally without OOM or container escape
4. Logs every action to the audit JSONL before any subsequent step
5. Completes a 10-file scan of a medium-sized C project under $50 in API costs
6. Passes the validation agent filter (final pass catches and drops false positives)
7. Produces a human-review package that a security researcher unfamiliar with the run can triage without additional context
